UX and General Improvements

  1. Add trade-direction (long/short/net) interactive filters in relevant pages such as Performance Analysis, Risk Factor Analysis, etc.
  2. Improve state management: Streamlit runs top-down every time a user clicks or navigates to a new page, causing previous user selections to be lost. Implement session state to preserve user selections across page navigation.
  3. Implement consistent cross-filtering functionality (similar to tools like Qlik and PowerBI). This enhancement depends on the improved state management mentioned above.
  4. Add Month-to-Date (MTD) and Year-to-Date (YTD) filter buttons in live portfolio dashboards to provide quick access to frequently monitored time periods.
  5. Ideally, well-designed dashboards act as self-service tools to go deeper into the underlying data and intermediate calculations, not merely top-level summaries. This can be achieved by integrating various inputs (eg: alpha scores) and other intermediate calculations used in the rebalancing or backtesting processes.
  1. Some improvement thoughts in the existing pages
  2. 1. Back Test Summary
    This page currently only shows summaries. It would be more valuable to display various time series graphs for analytics such as:
    • a) Volatility, b) Drawdowns, c) Tracking Error, d) Beta

  3. 2. Performance Analysis
    Add more Group By analysis options: For example, to better understand the contribution by each group (sectors/country), pie charts could be incorporated.

  4. 3. Risk Factor Attributions
    The current risk model used is simplistic, with only one group based on SEC industry classifications. Commercial risk models typically include distinct groupings such as:
    • Style Factors
    • Industry Factors
    • Country Factors, Currency Factors (Global Models)
    The visualizations should be flexible to allow users to select/filter these groups.

  5. 4. Rebalance Analysis
    Similar to the Performance Analysis improvements, additional Group By capabilities could be added.

  6. 5. Stress Test analysis
    For reporting needs, a new page with VaR analysis numbers and historical or hypothetical scenario P&Ls could be added.
